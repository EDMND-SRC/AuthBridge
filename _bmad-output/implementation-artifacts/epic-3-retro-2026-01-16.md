## Challenges & Issues

**Bob (Scrum Master):** "Now let's talk about challenges. Where did we struggle?"

**Dana (QA Engineer):** "The Mantine 8 API changes were painful. Table.ScrollContainer wrapper, Select data prop changes, DatePicker import changes - every component had breaking changes."

**Charlie (Senior Dev):** "That's fair. The upgrade guide helped, but we still had to fix dozens of components. Story 3.1 took longer than expected because of this."

**Winston (Architect):** "We should have done a spike first to understand the breaking changes before committing to the upgrade."

**Bob (Scrum Master):** "Edmond, what's your take on the Mantine upgrade challenge?"

**Edmond (Project Lead):** "It was the right call to upgrade - Mantine 8 has better performance and accessibility. But we could have planned better. A half-day spike would have identified the breaking changes upfront."

**Alice (Product Owner):** "From a product perspective, the upgrade didn't delay delivery. Story 3.1 absorbed the upgrade work, and subsequent stories went smoothly."

**Bob (Scrum Master):** "Another pattern I noticed - the E2E tests required careful selector management. Mantine components don't always have predictable selectors."

**Dana (QA Engineer):** "Yeah, we added data-testid attributes to everything. That's the right pattern, but it's extra work."

**Charlie (Senior Dev):** "The auth fixture was tricky too. We had to mock Cognito authentication for E2E tests. The pattern works now, but it took iteration."

**Bob (Scrum Master):** "Edmond, did you notice any other challenges during the epic?"

**Edmond (Project Lead):** "The webhook delivery in Story 3.3 initially had a TODO for SQS integration. The code review caught it, and we fixed it. But it shows we need to be careful about TODOs in production code."

**Key Challenges:**
- Mantine 8 breaking changes required extensive component updates
- E2E selector management needed data-testid attributes throughout
- Auth fixture for Playwright required Cognito mocking
- TODO comments in production code (caught in review)
- No spike before major dependency upgrade
