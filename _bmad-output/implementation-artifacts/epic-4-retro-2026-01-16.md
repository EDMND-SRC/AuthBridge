## Challenges & Issues

**Bob (Scrum Master):** "Now let's talk about challenges. Where did we struggle?"

**Charlie (Senior Dev):** "The DynamoDB Scan for API key validation is a concern. When validating an API key, we scan all keys to find a match. That's O(n) and won't scale."

**Winston (Architect):** "That's a valid concern. We need a GSI on the hashed API key for O(1) lookup. I added a TODO in the code."

**Bob (Scrum Master):** "Edmond, what's your take on the DynamoDB Scan issue?"

**Edmond (Project Lead):** "It's technical debt we need to address. For MVP with a few clients, it's fine. But before scaling, we need that GSI. Let's add it to Epic 5 or a prep sprint."

**Dana (QA Engineer):** "The integration tests for webhooks were tricky. We needed a mock webhook server to test delivery and retry logic. The pattern works now, but it took iteration."

**Charlie (Senior Dev):** "The multipart form data parsing in Story 4.3 was more complex than expected. Binary data handling in Lambda requires careful buffer management."

**Bob (Scrum Master):** "Any other challenges during the epic?"

**Alice (Product Owner):** "The session token expiry change from 24 hours to 30 minutes in Story 4.2 was a security improvement, but it required updating the SDK documentation."

**Winston (Architect):** "The API key format with environment prefix (`ab_live_`, `ab_test_`) was caught in code review. Good that we standardized it early."

**Key Challenges:**
- DynamoDB Scan for API key validation (needs GSI for scale)
- Mock webhook server setup for integration tests
- Multipart form data parsing complexity
- Session token expiry documentation update
- API key format standardization (caught in review)
